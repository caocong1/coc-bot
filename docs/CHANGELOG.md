# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

## [0.11.1] - 2026-03-16

### Fixed
- **`.ra` 错误提示细化**：`.ra 技能名` 查找失败时不再笼统返回"格式错误"，而是区分"未找到角色卡"和"角色卡中未找到该技能"两种情况，给出针对性提示
- **API 错误分类**：DashScope 客户端新增 `DashScopeApiError` 错误类，区分配额耗尽（403/AllocationQuota）和临时错误（429/5xx），KPPipeline 意图分类失败时按错误类型输出不同级别日志
- **指令提取容错**：所有结构化指令（`[TIME_ADVANCE:]`、`[SET_SCENE:]` 等 9 种）的正则增加冒号后空格容忍，防止 AI 加空格时指令泄露到玩家输出
- **开场推进过快**：OpeningDirector 骨架 prompt 限制 beat 数量（单人≤2、多人≤3），beat 文本 prompt 强制 100-250 字聚焦氛围、不推进调查，硬上限从 5 beat 降到 3

### Changed
- **AI KP 信息节奏控制**：system prompt 新增"每次回复最多推进 1 个调查节点"规则，避免开场一次性倾倒多段调查信息
- **引号=对话规则**：明确引号内内容为角色台词、无引号内容为行动/内心活动，NPC 不对内心独白做出反应
- **NPC 知识边界**：NPC 不可读取 PC 角色卡元数据（职业/技能值等），除非 PC 主动透露或模组设定
- **职业适配剧情**：AI KP 根据 PC 职业个性化剧情切入点（作家→采访线索，医生→病患求诊等）
- **RP 与检定平衡**：好的角色扮演可酌情降低难度或给予奖励骰，日常无风险社交好 RP 可直接成功
- **KP 引导信息隔离**：模组中写给 KP 的指引/备注段落严禁直接给玩家，条件句只有满足后才执行
- **全中文输出**：system prompt 明确要求全中文叙事，不混入英文单词

## [0.11.0] - 2026-03-16

- **房间详情支持修改模组**：玩家端房主和管理端管理员现在都可直接在房间详情页切换模组；仅等待中房间可改，切换后会自动覆盖为新模组默认筛卡条件，但保留成员、已绑定角色卡和人物关系
- **前端主题变量统一**：Web 前端已将残留的 `var(--surface)`、`var(--border)`、`var(--text)` 等旧主题变量写法统一替换为 `--color-*` 规范变量，避免部分页面样式依赖兼容别名而出现失效或不一致
- **首页不再自动跳玩家端**：根路径 `/` 不会再因为本地残留 `player_token` 而直接进入玩家端；只有访问 `/player` 或登录链接里显式带 `token` 时才会进入玩家侧
- **管理端认证失效回首页**：管理端 API 请求若返回 401，现在会清理本地 `admin_secret` 并直接跳转回首页，避免继续停留在失效的管理页面
- **超时默认值调长**：AI 流式调用默认超时从 60 秒提高到 120 秒，房间审卡默认超时从 10 分钟提高到 20 分钟；两者都可通过 `.env` 中的 `DASHSCOPE_STREAM_TIMEOUT_MS` 和 `ROOM_REVIEW_TIMEOUT_MS` 覆盖
- **`.jrrp` 语气收口**：今日人品评语整体改得更偏幽默安慰，低分时减少纯倒霉/纯打击语气，改成更常见的“别硬上 / 宜保守 / 先避坑”式表达
- **`.jrrp [话题]`**：支持在 `.jrrp` 后追加自由文本作为评语参考主题，例如 `.jrrp 跑团`、`.jrrp 上班`、`.jrrp 恋爱`；带后缀的调用不进入当天缓存，但分数仍保持每日固定
- **`.jrrp` 后缀清洗**：带后缀的话题仍然只作为 AI 的自然参考提示，不做词表分类；同时会裁剪超长后缀，避免提示词被无关长文本污染
- **`.jrrp` 改为号码语义优先**：AI prompt 不再把 1~100 简单按高分/低分分档，而是要求围绕“这个具体数字的独特意味”来写；同样接近的数字也应有不同味道

## [0.10.0] - 2026-03-15

- **角色卡空白 Excel 下载**：玩家端“我的角色卡”页在“导入 Excel”旁新增“空白卡下载”，会通过带鉴权的玩家接口直接下载仓库根目录的 `[充实车卡版本]空白卡.xlsx`
- **玩家参考资料去脏行**：玩家端参考资料的防具表和载具表会自动过滤 Excel 尾部混入的“术语解释”说明行；抽取脚本也同步补上清洗规则，避免重新生成 JSON 时再次把说明行写进玩家可见数据
- **房间人物关系重构**：房间人物关系从“按 QQ 两两设置”改为“按当前房间已绑定的 PC 卡建立群组关系”；支持 2 人或更多人共用一条中文关系名，旧英文关系会在首次读取时尽量迁成中文关系组
- **人物关系主路径收口**：`.room relation` 已从房间命令中移除；人物关系统一在 Web 房间详情页配置，玩家端和管理端都改成多选当前 PC 卡、填写中文关系名与备注
- **玩家端操作手册更新**：Web 端“操作手册”与入门文档已改成说明“人物关系在房间详情页配置”，不再展示旧的 `.room relation list/set/clear` 流程

## [0.9.0] - 2026-03-15

### Changed

- **公开团 / 秘密团分流**：模组规则包新增 `playPrivacyMode` 与 `privacyNotes`；默认所有模组按公开团处理，只有明确要求秘密推进的模组才启用私聊与秘密分线
- **导演表现层收口**：公开团开场不再向玩家输出 `【镜头：...】`、不再自动提示开场频道分配，导演标记仅保留为内部事件
- **PRIVATE_TO 收口**：公开团中若模型仍输出 `[PRIVATE_TO:...]...[/PRIVATE_TO]`，内容会自动降级回公开叙事并记录 warning；只有秘密团才真正发送私聊
- **`.scene` 降级为高级工具**：默认公开团只使用 `main` 频道，`.scene` 主要保留给管理员/KP 手动分线和秘密团使用
- **管理端规则包补充隐私模式**：模组详情页可直接查看和修改“公开团 / 秘密团”推进模式与隐私说明，避免误判秘密团
- **房间即跑团**：玩家侧统一收敛为“一个房间就是一场跑团”，房间详情现在承载成员、绑卡、审卡、运行状态、游戏时间和消息历史；`kp_sessions` 退回内部运行时实现
- **旧“我的团”入口收口**：玩家端移除独立“我的团”导航，旧 campaign 链接会自动跳转到对应房间的消息历史；找不到房间时回到房间列表并提示已归档
- **管理端房间详情重构**：管理端房间详情改为与玩家端共用主体信息结构，仅额外增加“管理区”承载 KP 设定、房间控制和运行时诊断
- **导演策略内收**：房间级“导演偏好”已从玩家端和管理端移除；开场导演改为使用系统内部全局策略，默认允许分头开场、自动判断开场风格、允许高强度模组内扩写，私密引子维持轻量
- **模组可见性过滤**：`.mod` 和玩家端模组列表不再显示仍有文档处于导入中的模组，避免把未就绪模组暴露给开团流程
- **玩家可见模组简介收口**：模组导入时的自动简介改为只允许写一条单句开场钩子，只保留最初邀请、委托、怪事或调查动机；玩家 Web 和 `.mod` 也会对已有简介再做一次防剧透收口，避免把中后段真相、怪物揭示或时间循环等内容直接暴露给玩家
- **DashScope 图像生成修正**：`qwen-image-2.0-pro` 改为调用同步 `multimodal-generation` 接口，不再误用异步 `text2image` 接口导致 DOCX 导入自动配图全部报 `url error`
- **模组场景图片聚合**：管理端模组详情的“场景图片”现在会同时显示上传/手工生成的图片，以及从 DOCX 文档中提取或补生成后写入 `ImageLibrary` 的图片；旧导入会通过 manifest 回溯关联，新的模组文档上传则直接绑定 `scenario_module_files.id`
- **模组删除级联清理**：删除模组或删除模组内文档文件时，现在会一并清理对应的导入产物，包括 `manifest.json` 条目、raw/chunks/keeper chunks 文本、文档提取/补图的图片库记录与物理文件，避免知识库残留孤儿文件
- **模组图片路径稳固化**：管理端读取模组图片、图片库和 manifest 不再依赖当前工作目录；同时管理端图片 URL 会自动附带 admin secret，浏览器 `<img>` 请求也能通过鉴权
- **模组详情独立页**：管理端模组列表新增“详情页”入口，模组文档、场景图片、结构化实体、关键物品和规则包可在独立页面完整查看，不必挤在折叠卡片里
- **模组列表收口**：管理端模组列表不再同时保留“展开详情”和独立详情页两套交互，列表仅显示概览，查看/编辑统一进入模组详情页
- **模组详情页编辑**：管理端模组详情页增加内联编辑入口，列表去掉展开后仍可直接修改名称、简介、时代、职业限制和总点要求
- **文档补图手动重生**：管理端模组详情中，自动补出的 AI 场景图不再只能提示“重新导入原文档”，现在可直接按当前提示词手动重新生成；文档提取图仍保持只读说明
- **模组图片放大预览**：管理端模组详情的场景图片支持点击放大查看，预览层会展示来源、说明和关闭提示，便于检查 AI 补图与文档提取图细节
- **玩家 token 过期回首页**：玩家端所有带鉴权的 API 请求在收到 401 后都会清理本地 token 并直接回到首页，不再停留在失效的房间/角色卡页面
- **场景图片卡可读性优化**：管理端模组详情中的场景图片卡放宽为更大的网格单元，标题支持多行显示，来源文档与说明拆层展示并提升可见行数
- **模组图片实时刷新**：`ImageLibrary` 读取时会自动从磁盘刷新，文档导入或自动补图完成后无需重启后端，模组详情刷新页面即可看到新图片
- **`.room pc` 自动认房间**：如果当前群里你只参与了一个未绑定群号的等待房间，`.room pc <角色卡名>` 会自动绑定到当前群并完成角色卡设置；同时支持显式写成 `.room pc <房间ID> <角色卡名>`
- **角色卡编辑回填**：编辑已有角色卡时，表单会自动拉取并回填原始职业、属性、技能分配、装备、背景、法术和经历数据，不再从默认空白值开始
- **房间详情操作收口**：玩家端房间详情移除了无实际作用的“开始审卡”和“复制邀请指令”按钮，避免把审卡入口和邀请流程误导成可在该页直接完成
- **模组/房间审卡约束**：模组不再使用“最低属性要求”，改为可选的“主属性总点要求”；新建模组默认 460，房间会继承该值且允许单独修改或留空
- **审卡校验**：房间审卡与后台警告改为校验 `STR/CON/SIZ/DEX/APP/INT/POW/EDU` 八项主属性总和必须等于房间要求
- **模组职业限制表单**：管理端模组编辑从自由文本改为基于标准职业库的可搜索多选，避免手输职业名带来的筛选误差
- **筛卡提示 UI**：模组编辑、创建房间和房间详情页补充了更明确的职业限制 / 总点要求说明，强调“房间筛卡只在当前房间生效，不会修改角色卡本身”
- **房间选卡即时提示**：房间详情页的角色卡列表现在会直接标出“符合要求 / 不符合”，并显示具体原因（如职业不符、总点不符）
- **房间内直接改卡**：房间详情页的选卡列表为每张卡增加了“编辑角色卡”入口，玩家看到不符合提示后可直接去修改并返回重选
- **房间详情页消息历史**：跑团消息从底部小区域改为独立 tab，大幅扩大显示区域，并在进入/刷新时自动滚到最新消息
- **消息历史气泡样式**：房间详情页的消息历史改为气泡式阅读布局，KP 与玩家消息使用不同对齐和底色，更容易快速分辨
- **房间详情视觉优化**：房间详情的 tab 导航改为更明显的分段切换器，消息气泡增加头像圆标、系统消息样式和更强的层次感
- **房间详情仪表盘化**：页头改为更清晰的概览面板，集中展示状态、模组、时间、成员数与快捷操作；成员列表也升级为更像人物卡的展示样式
- **房间信息页统一面板化**：成员、选卡、人物关系等主区块统一为同一套卡片层次和说明样式，玩家端不再混入导演偏好配置
- **成员列表合规徽章**：房间详情左侧成员列表也会直接标出当前所选角色卡是否符合房间职业/总点要求，并显示需调整原因
- **消息历史时间分隔**：长跑团记录现在会按日期自动插入分隔条，并在较长时间跳跃处显示时间间隔提示，阅读连续性更好
- **消息重点高亮**：房间消息会对检定结果、线索推进、时间推进等常见内容自动加上类型徽标，长记录更容易扫读
- **成员卡摘要增强**：成员卡增加主属性总点与职业摘要，并根据“已准备 / 需调整 / 已绑定”状态显示更明确的视觉层次
- **当前绑定角色卡摘要卡**：房间详情会单独展示玩家当前绑定的角色卡、合规状态与快捷操作，不必在列表里反复定位
- **重点消息头部样式**：检定结果、线索推进、时间推进等消息会显示更明显的标题条和状态标签，长跑团记录更容易快速扫视
- **检定消息结果卡**：检定与理智类消息会按“结果头 / 明细 / 警告”拆层显示，长消息不再挤成一整块文本

## [0.8.0] - 2026-03-15

### Added

- **开场导演**：`OpeningDirector` 取代旧的三段式固定开场，改为“skeleton + 并行 beat 文本”两步生成，可基于房间人物关系、导演偏好、模组资产和规则包生成分镜式开场
- **推进导演**：新增 `SessionDirector` 与 `director_cue` / `director_seed_resolved` 事件，在连续无进展交互后给 KP 注入低侵入的推进方向
- **房间关系与导演偏好**：新增 `campaign_room_relationships` 表和 `campaign_rooms.director_prefs_json`，支持开场关系预设、分头开场偏好、扩写强度和私密引子偏好
- **玩家入口扩展**：新增 `.room pc`、`.room relation list/set/clear`，玩家端 API 与房间详情页可读写关系和导演偏好

### Changed

- **开场落库**：开场计划、镜头标识和公开/私密开场文本都会进入 `kp_events`，恢复会话时可重建 opening plan、初始频道分配和导演种子
- **上下文注入**：`ContextBuilder` 新增导演层，支持在 cue 触发时向 KP 注入当前汇合目标、未解决开场种子、最近导演提示和分频道状态
- **房间详情页**：Web 玩家端增加轻量关系编辑区和导演偏好表单，便于在开团前调整开场组织方式

## [0.6.0] - 2026-03-15

### Added
- **AI KP 事件流底座**：新增 `kp_events` 作为跑团运行历史表，`SessionState` 支持事件回放重建快照
- **SceneChannel 场景频道**：支持 `focusChannel`、玩家频道归属、紧急频道标记和 `.scene` 命令
- **私聊 sidecar**：AI KP 可通过 `[PRIVATE_TO:调查员]...[/PRIVATE_TO]` 指令向特定玩家发送私密信息

### Changed
- **ContextBuilder**：按当前频道过滤角色卡与消息历史，旧历史摘要改为事件派生摘要缓存
- **KPPipeline**：新增场景/线索/私密指令抽取，输出合规兜底，并把检定请求落回 `PendingRoll`
- **CampaignHandler**：从单队列升级为按频道排队，非焦点频道消息不丢失，紧急事件会提示切换焦点

### Fixed
- **私聊 sidecar 投递**：`CampaignOutput.privateMessages` 现已在 bot 发送层真正走 QQ 私聊发送，并带失败告警
- **私聊失败群内回退**：所有经 `server/index.ts` 发出的私聊，在失败时都会向对应群发送错误提示，避免静默丢失
- **私聊节流与统一投递**：开团/恢复、Web 自动开团和 `.room` 自动开团现统一走 `CampaignOutput` 投递器，批量私聊之间增加 800ms 节流
- **时间屏障**：存在未处理频道中断时，`TIME_ADVANCE` / `SET_TIME` 的大跨度时间推进会被限制并写入警告
- **双写原子性**：核心 dual-write mutator 改为事务化，事件表与 legacy 表不会再出现半提交
