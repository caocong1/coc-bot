## 四、已实现功能概览

### 本次新增（2026-03-12）

#### 跑团房间 & QQ 指令体系

- **房间不绑定群**：`campaign_rooms.group_id` 为 nullable，创建时不填群号
- **`.room` 指令**（完整生命周期）：
  - `create` — 创建房间，可绑定模组 ID
  - `join` — 加入房间，私发带 token 的 Web 车卡链接
  - `start` — 在当前群开始跑团（仅创建者）
  - `pause / resume / stop` — 暂停/恢复/结束当前群的 session
  - `list / info` — 查看我的活跃房间 / 房间详情
- **`.mod` 指令**：`list [页]` / `info <ID>` — 在 QQ 内浏览模组
- **`.web login` 私聊限定**：群内发送返回提示去私聊，避免 token 泄露

#### 模组管理系统

- **DB**：`scenario_modules` 表（id / name / description / era / allowed_occupations / min_stats）+ `scenario_module_files` 表（关联模组的文档和图片，含 import_status）
- **后端 API**（`/api/admin/modules/*`）：
  - CRUD 模组（创建/查看列表/查看详情/编辑/删除）
  - 上传文档（PDF/DOCX/TXT）→ 后台触发建索引，状态写回 DB，前端自动轮询
  - 上传图片 / AI 生成图片（存储在 `data/knowledge/images/modules/{moduleId}/`）
  - 删除文档/图片
- **玩家 API**（`/api/player/modules`）：列出所有模组（含约束，不含文件详情）
- **前端 Admin**：`ScenarioManager`，模组卡片展开显示文档列表（上传/删除/状态自动刷新）+ 图片区
- **前端 Player**：「模组列表」页；创建房间时模组下拉选择 + 约束预览

#### Web 车卡两步向导
- **Step 1**：掷出 4 组属性对比表，玩家选一组
- **Step 2**：属性只读展示，选择职业后显示职业技能点/兴趣点预算，分配技能，填背景故事
- 提交调用 `playerApi.createCharacter()` / `updateCharacter()`，自动同步 bot DB

#### 启动稳定性
- `data/bot.pid` 管理：启动时自动 kill 旧实例，退出时清理
